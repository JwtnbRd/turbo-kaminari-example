# タスク完了時のチェックリスト

## 🚨 最重要: 品質確認コマンド実行

### バックエンド（Rails）
```bash
# テスト実行
./scripts/container-backend.sh test
# または
bundle exec rspec

# リント・フォーマット確認
bundle exec rubocop
bundle exec rubocop -a  # 自動修正

# セキュリティチェック
bundle exec brakeman
```

### フロントエンド（React）
```bash
# TypeScript型チェック
npm run typecheck
# または
tsc --noEmit

# リント・フォーマット
npm run lint
npm run format

# テスト実行
npm run test

# ビルド確認
npm run build
```

## コード品質チェック

### 必須確認項目
- [ ] 全テストが成功している
- [ ] リントエラーがない
- [ ] TypeScriptエラーがない（フロントエンド）
- [ ] セキュリティ問題がない
- [ ] パフォーマンス問題がない

### コードレビュー（自己チェック）
- [ ] 命名が適切か
- [ ] コメントが必要な複雑な処理にコメントがあるか
- [ ] 不要なコードが残っていないか
- [ ] ハードコーディングがないか
- [ ] エラーハンドリングが適切か

## 機能確認

### 動作テスト
- [ ] 実装した機能が正常に動作する
- [ ] エラーケースも適切に処理される
- [ ] 既存機能に影響がない（リグレッションテスト）

### UI/UX確認（フロントエンド）
- [ ] レスポンシブデザインが正しい
- [ ] ローディング状態が適切
- [ ] エラー表示が分かりやすい
- [ ] アクセシビリティ要件を満たす

## ドキュメント更新

### 必要に応じて更新
- [ ] README.md
- [ ] API仕様書
- [ ] 設計ドキュメント
- [ ] 運用手順書

## Git管理

### コミット前確認
- [ ] 適切なコミットメッセージ
- [ ] 不要なファイルが含まれていない
- [ ] .gitignoreが適切に設定されている

### ブランチ管理
- [ ] feature/bugfix/hotfixのプレフィックスが適切
- [ ] ブランチ名が作業内容を表している
- [ ] developブランチから作成されている

## 🛡️ セキュリティチェック

### 重要確認項目
- [ ] 秘匿情報（API Key等）がコードに含まれていない
- [ ] 環境変数が適切に設定されている
- [ ] バリデーションが適切に実装されている
- [ ] 認証・認可が正しく動作している

## パフォーマンス

### 確認項目
- [ ] N+1クエリが発生していない（バックエンド）
- [ ] 不要なレンダリングが発生していない（フロントエンド）
- [ ] 画像サイズが適切
- [ ] バンドルサイズが妥当

## 最終確認

### 本番環境での動作確認（該当する場合）
- [ ] 本番環境で正常に動作する
- [ ] 本番データで問題がない
- [ ] パフォーマンスが許容範囲

### チーム連携
- [ ] レビュー依頼
- [ ] 関係者への通知
- [ ] ドキュメント共有

---

**注意**: このチェックリストはタスクの複雑さに応じて調整してください。簡単な修正では一部を省略可能ですが、重要な機能追加では全項目を確認することを推奨します。
